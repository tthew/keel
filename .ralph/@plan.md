- **Counter-file stub pattern needs empty-string guard (iter-215 LESSON):** When writing a state-machine docker stub that progresses through values via a counter file, `: > counter` truncates to empty (not "0"); `cat ""` then yields empty string; `case "" in 0)` does NOT match (matches `*)` instead) → stub returns the default-arm value on first call, not the 0-indexed first value. Fix: `count=$(cat counter 2>/dev/null || true); count=${count:-0}` + initial `rm -f counter` (no file = default 0).
