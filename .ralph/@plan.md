## Notes

- **iter-174 zsh-vs-bash mapfile gotcha:** the Bash tool defaults to zsh, which lacks `mapfile`. First parity-smoke run silently dropped fragment files because `mapfile -t fragments < <(find …)` errored as "command not found" and the fragments array stayed empty — composition emitted only baseline + local. Re-ran under explicit `bash -c '...'` to fix. Apply at every future iteration-env shell-script smoke that uses bash 4.4+ builtins (`mapfile`, `readarray`, associative arrays via `declare -A`): wrap in `bash -c '...'` rather than relying on the implicit shell. Promoted to RALPH.md § Lessons.
- **iter-174 `BASH_SOURCE[0]==$0` main-guard sourceability pattern:** wrapping the dispatcher in `if [[ "${BASH_SOURCE[0]}" == "${0}" ]]; then ... fi` makes a shell script sourceable without firing the dispatcher — when sourced, BASH_SOURCE[0] is the script path while $0 is the parent shell's $0, so the guard skips the main block. Applied in `whitelist.sh` to enable Subtask 7.4 parity harness without code duplication. Apply at every future bash-script story where a parity / unit / harness test wants to invoke functions in isolation. Promoted to RALPH.md § Lessons.
- **iter-174 Edit tool `replace_all=true` substring-overlap gotcha:** the Edit tool's `replace_all=true` on `- [ ] **Task 1` matches BOTH "Task 1" AND "Task 10/11" because "Task 1" is a substring. Caused all three checkboxes to flip in one operation, then individual subsequent flips for Tasks 10 and 11 errored "string not found" (already done). Apply at every future batch-checkbox-flip on enumerated items: anchor the trailing context (e.g., `- [ ] **Task 1 —`) to disambiguate substring overlap. Promoted to RALPH.md § Gotchas.
- **iter-174 SC vs subtask wording discrepancy — `mv` vs `cp` atomicity:** Subtask 4.5 proposed `cp -f "${tempfile}" "${WHITELIST_LOCAL}"` then `rm -f "${tempfile}"` for atomic append. SC-9 (the binding contract) mandates `mv` for true atomic replacement (rename(2) is atomic on same FS; `cp` + `rm` is not). When subtask wording diverges from SC-level binding contract, follow the SC. Implementation uses `mv` for atomic replacement; flagged in Completion Notes. Promoted to RALPH.md § Lessons.
