- **iter-273 NOVEL LESSON (compose env-propagation sites are normalisation chokepoints):** any `docker-compose.yml` env var sourced from an `.envrc` knob via `${VAR:-default}` interpolation MUST source the RESOLVED host-side value (`${VAR_RESOLVED:-default}`) when there's an adjacent resolver that case-folds or otherwise normalises the knob. Compose interpolation is a dumb substitution at host parse time — it does NOT invoke resolver functions. Symptom pattern: resolver case-folds the COMPOSE-FILE-ARG decision (`KEEL_DEVBOX_COMPOSE_FILE_SSH` set correctly), but the PROPAGATED env value stays raw, producing drift (`True`/`TRUE` → port published but no sshd). Generalises to every future resolver-gated knob (Story 2.11 `KEEL_DEVBOX_SHARED`, Story 2.10 `KEEL_DEVBOX_PREREQ_*`, future egress/whitelist gates).
