# packages/devbox/whitelist/github-rotating.txt — Story 2.3 (Task 2.2) + Story 2.18 (SC-2)
#
# GitHub API + git-clone + ghcr.io allow-list fragment. Covers `gh` CLI
# (Story 2.9 OAuth), git clone / fetch / push over HTTPS, raw file pulls
# used by Dockerfile install scripts and Renovate metadata lookups, and
# ghcr.io container pulls (e.g. pg_uuidv7 image tag per Story 1.15).
#
# Filename suffix `-rotating.txt` is the Story 2.18 SC-2 annotation contract:
# every domain in this fragment is classified `rotating` by the whitelist
# composers (`compose_whitelist()` / `compose_whitelist_into()`) and emitted
# into the `.classification` sidecar with type=rotating. Per Story 2.18 AC1,
# rotating-flagged domains drive `nftset=` directive emission in the rendered
# `dnsmasq.conf` so the kernel-side `inet keel_egress` named sets `gh_v4` /
# `gh_v6` (declared in `nftables/egress.nft`) self-fill as DNS rotates upstream.
