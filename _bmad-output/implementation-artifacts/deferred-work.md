## Deferred from: code review of story-1.8 (2026-04-20)

- **sourcePath path-traversal / format guard in Zod schema.** _Source:_ Blind Hunter + Edge Case Hunter. _Location:_ `packages/keel-invariants/src/invariants.manifest.ts:6` (`sourcePath: z.string().min(1)`). _Nature:_ schema accepts absolute paths, `..` traversal, Windows backslashes, leading slashes. - **`id` uniqueness across array not enforced by schema.** _Source:_ Blind Hunter + Edge Case Hunter. _Location:_ `packages/keel-invariants/src/invariants.manifest.ts:11` (`InvariantsSchema = z.array(InvariantSchema)`). _Nature:_ Zod's `z.array` permits duplicate elements; two entries with the same `id` would silently validate. If a future authoring error broke that invariant (same `sourcePath`, different `contentHash`), the schema wouldn't catch it. _Carry-to:_ Story 1.9 walker re-computes `sha256` per distinct `sourcePath` and surfaces cross-entry mismatch immediately.
- **`invariants` array immutability — `readonly` / `Object.freeze`.** _Source:_ Blind Hunter + Edge Case Hunter. _Location:_ `packages/keel-invariants/src/invariants.manifest.ts:89` (`export const invariants: Invariant[]`). _Nature:_ consumer of `@keel/keel-invariants` receives a live reference to a mutable array; a misbehaving consumer (`invariants.push(...)` / `invariants[0].id = '...'`) would corrupt the module-level singleton for every other importer. Spec-silent hardening; low-cost but out-of-scope for contract-only 1.8.
- **Schema-evolution metadata on `Invariant` entries (`schemaVersion` / `deprecated` / `since`).** _Source:_ Blind Hunter. _Location:_ `packages/keel-invariants/src/invariants.manifest.ts:3-9` (`InvariantSchema` object shape). _Nature:_ no first-class way to mark an invariant as deprecated-but-not-yet-drifted, no schema version field on the root manifest, no `since` / introduced-in-story field per entry. _Carry-to:_ Story 1.9's enforcement side will need "deprecated ≠ drift" semantics at minimum (otherwise sunsetting a rule and removing its registration both look like drift). ## Deferred from: code review of story-1.9 (2026-04-20)

_Story 1.9 (Invariant sync-gate runtime tooling — reader/walker/drift-detector) — iter-7 `/bmad-code-review (args: "2")`. The 10 defers below are all defensive-hardening candidates natural-scope for Story 1.10+ substrate refactors OR the Story 1.16 test-runner landing; none are exploitable in the current single-consumer, linux-only, text-sources-only baseline._

- **Shallow `Object.freeze` vs "readonly + Object.freeze" spec claim.** _Source:_ Edge Case Hunter. _Location:_ `packages/keel-invariants/src/invariants.manifest.ts:~130`. _Nature:_ `Object.freeze(invariants)` locks the top-level array but entries and their `anchors` sub-arrays remain mutable at runtime (`invariants[0].id = 'HACKED'` succeeds; `invariants[0].anchors.push('X')` succeeds; `invariants.push(fake)` throws because only the outer array is frozen). `readonly Invariant[]` is a compile-time TS signal with zero runtime effect. Task 3 (Story 1.9 CR defer #5 from Story 1.8) advertised "readonly + Object.freeze" as a runtime guarantee; the landed implementation is the shallow-only variant. Not exploitable in-process today (no consumer mutates entries), but the spec over-advertises the guarantee. _Carry-to:_ either Story 1.10+ deep-freeze refactor (`entry + entry.anchors` Object.freeze) OR amend spec text to "shallow freeze (array-reassignment guard only)". Two-line code change either way.
- **`check.ts` lacks try/catch around `runSyncGate` — raw ENOENT stacktrace on missing INVARIANTS.md.** _Source:_ Edge Case Hunter. _Location:_ `packages/keel-invariants/src/check.ts` (top-level await). _Nature:_ if `INVARIANTS.md` is absent (cwd mismatch, dist-layout drift, moved-file) or `repoRoot` resolution misfires, Node emits `UnhandledPromiseRejection` + raw ENOENT stacktrace. AC 6's intent is "non-zero reliably fails the workflow AND the drift report renders in CI logs" — raw stacktrace exits non-zero but violates the "renders a report" half. _Carry-to:_ wrap the top-level await in try/catch, emit `{status: 'error', message: e.message}` and exit 1. Belongs with Story 1.16 CI-wiring story where the error-surface is tested end-to-end in GitHub Actions logs.
- **Traversal refine gaps in `sourcePath` Zod validation.** _Source:_ Edge Case Hunter. _Location:_ `packages/keel-invariants/src/invariants.manifest.ts:~10` (`refine(p => !p.startsWith('/') && !p.startsWith('\\\\') && !p.includes('..') && !p.includes('\\\\'))`). _Nature:_ (a) `foo/bar/%2e%2e/etc/passwd` passes (percent-encoded `..` — benign today since `readFile` doesn't URL-decode, but the refine message implies broader guard); (b) `D:/Windows` passes (Windows drive-absolute — low-risk on Linux CI, non-zero on Windows contributors); (c) null-byte / newline / CR characters not rejected; (d) symlinks defeat the refine at the FS layer regardless (`readFile` follows symlinks, so a committed `src/foo` → `/etc/passwd` symlink hashes the foreign content). _Carry-to:_ tighten refine (`!/\0|\n|\r/.test(p)`, `!/^[a-zA-Z]:/.test(p)`) + use `fs.lstat` + reject `S_IFLNK`. Natural Story 1.10+ schema-hardening pass.
- **UTF-8 binary-file mangling in `manifest-reader.ts`.** _Source:_ Edge Case Hunter. _Location:_ `packages/keel-invariants/src/manifest-reader.ts:8` (`readFile(absPath, 'utf-8')`). _Nature:_ reading bytes as utf-8 string and hashing the decoded string diverges from byte-level sha256 on any non-UTF-8 content (binary files, files with invalid sequences). All current invariants source text files; vacuous today. But a future invariant targeting a binary file (lockfile digest, wasm module) would hash inconsistently with `sha256sum` CLI output. _Carry-to:_ swap to `readFile(absPath)` → Buffer, then `createHash('sha256').update(buffer).digest('hex')` — aligns byte-for-byte with `sha256sum`. One-line change.
- **`Promise.all` error-swallow at `sync-gate.ts:47-57`.** _Source:_ Edge Case Hunter. _Location:_ `packages/keel-invariants/src/sync-gate.ts:47-57` (catch-all returning `null` inside `Promise.all`). _Nature:_ all non-ENOENT failures (permission denied, I/O error, path-is-a-directory) collapse to `removed-from-source-only` drift. Misleading root-cause analysis in CI. _Carry-to:_ narrow catch to `err.code === 'ENOENT'`; re-throw others OR emit `error`-kind drift. Natural hardening alongside the check.ts try/catch defer above.
- **Anchor-parser O(n·m) complexity.** _Source:_ Edge Case Hunter. _Location:_ `packages/keel-invariants/src/sync-gate.ts` (single-pass regex over doc per entry). _Nature:_ linear in entries × doc lines; at current scale (10 entries × ~60 doc lines) sub-millisecond; at 500 entries × 500 lines still sub-second. Not a live risk; flagged for principled scale-awareness. _Carry-to:_ no action needed for 1.x; if manifest grows past ~500 entries or doc past ~5000 lines, revisit with a single-pass map-building approach.
- **Duplicate-id refine message shape.** _Source:_ Edge Case Hunter. _Location:_ `packages/keel-invariants/src/invariants.manifest.ts:~22-28` (id-uniqueness `.superRefine`). _Nature:_ triplicate IDs emit 2 separate issue messages (one per extra occurrence), not one-per-pair. - **Whitespace-in-anchor backtick normalisation.** _Source:_ Edge Case Hunter. _Location:_ `packages/keel-invariants/src/sync-gate.ts` (anchor regex). _Nature:_ a hypothetical anchor line with trailing whitespace inside the backticks (`` `INV-foo-bar ` `` vs `` `INV-foo-bar` ``) would be treated as distinct IDs. ### Classification methodology (`/bmad-code-review (args: "2")` — "Create action items")

Three parallel adversarial layers per `.claude/skills/bmad-code-review/steps/step-02-review.md`: Blind Hunter (`bmad-review-adversarial-general`; diff-only, no project context), Edge Case Hunter (`bmad-review-edge-case-hunter`; diff + repo read access; path-enumeration method), Acceptance Auditor (diff + spec file + context docs; AC-violation-only).

Triage bucketing per `step-03-triage.md`:

- `patch` — fix unambiguous, no human input needed.
- `defer` — real issue, not actionable now (e.g. scope carve-out, spec-silent hardening).
- `dismiss` — false positive, by-design, or handled elsewhere.
- `decision_needed` — ambiguous choice (not applicable when auditor found zero AC violations).

Under adversarial-triage-is-default (per `.ralph/PROMPT_build.md § Story Lifecycle: "EVERY CR action item becomes a QUEUE fix task unless IP records defer: <reason> per item. Adversarial triage is the default."`): every non-dismissed finding must either get a `patch` fix task queued or an explicit `defer: <reason>` recorded. Story 1.8's 6 defers above are all either Story 1.9-enforcement-side scope or spec-silent hardening — no patches needed for the `sm-verified → done` transition.
