## Lessons learned

Things that went wrong, and why — so the next Ralph doesn't repeat them.

- 2026-04-19: IP can drift badly between mini-epics when the user manually swaps branches outside a Ralph iteration. First action on every iteration: reconcile `.ralph/@plan.md` against the actual branch/PR state (`git branch --show-current`, `gh pr view`) before treating the IP NOW as authoritative. Stale DONE sections are harmless; a stale NOW is misleading. Concrete reconcile for worktree-based runs: the main worktree owns `main`, so the Ralph worktree cannot `git checkout main` — use `git checkout -b <new-branch> origin/main` after `git fetch origin main`.
- 2026-04-19: Epics and architecture can disagree on scope. Architecture.md "First Implementation Priority" (lines 1395–1442) bundles Story 1.1 + devbox absorption + Ralph harness + keel-invariants seed + docs/research + all 14 package shells — far beyond Story 1.1's narrow AC in epics.md (just the monorepo + TS refs). When this happens, epics.md AC wins (it's the sprint-decomposed source of truth); record variances in the story's Project Structure Notes so dev-story knows what to defer.
- 2026-04-19: **TS5090 in per-package tsconfigs.** `pnpm -w typecheck` (Task 7) fired `TS5090: Non-relative paths are not allowed when 'baseUrl' is not set` at every per-package `tsconfig.json(3,3)` — the error is inherited-paths-flatten-into-extending-file. Root cause: `tsconfig.base.json` declared `paths` with bare values (`"packages/db/src/index.ts"`) and no `baseUrl`. TS 5.0+ accepts `paths` without `baseUrl` ONLY when each value is relative (leading `./`). Fix: prefix every `paths` value with `./`. Chose that over `baseUrl: "."` so bare-import resolution (via `moduleResolution: bundler` + package.json) stays independent of the paths map. When adding a new `@keel/<pkg>` entry to `tsconfig.base.json` in later stories, remember the `./` prefix. **Follow-up for Story 1.2 Task 1:** after moving `tsconfig.base.json` two dirs deeper into `packages/keel-invariants/`, every `paths` value needs its `./` recomputed as `../<pkg>/src/index.ts` (or `../../apps/web/src/index.ts`) — still relative, still TS5090-safe, but two path segments shift.
- 2026-04-19: **Multi-commit story PRs drift PR metadata from reality.** The first commit on `feat/story-1-1-...` was `docs(story): create Story 1.1 spec` and Ralph created PR #217 at that moment with a docs-only body. 8 task commits later, the body still said "This PR is docs-only; no code" — misleading for reviewers. - 2026-04-19: The `WIP` GitHub Marketplace app sets a commit status that stays `IN_PROGRESS` while a PR is Draft and resolves green the instant the PR is marked ready. `gh pr checks --watch --fail-fast` on a Draft PR whose only check is WIP will hang indefinitely. - 2026-04-20: **Claude's Bash tool runs zsh; `${PIPESTATUS[0]}` comes back empty.** In zsh, the array is `pipestatus` (lowercase, 1-indexed) and `PIPESTATUS` is unset. Observed repeatedly in Story 1.6 Task 2 probes when wrapping `git commit … | tee …` in a subshell. **Workaround:** rely on independent evidence instead — `git log -1 --oneline` (unchanged tip == commit aborted), `git status --short` (file still staged == commit aborted), `Passed/Failed` line in prek output.
